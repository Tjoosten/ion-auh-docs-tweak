Installation

  • 1. Download the latest version.
  • 2. Copy the files from this package to the corresponding folder in your application. For emxample, copy Ion-auth/config/ion_auth.php to system/application/config/ion_auth.php.
  • 3. Tun the appropriate SQL file from the /sql directory.

The default login is:

  • Email: admin@admin.com
  • Password: password

Upgrading

  • 1. Download the latest verion.
  • 2. Overwrite "libraries/ion_auth/php" and "models/ion_auth_model.php" with the new versions.

Loading Ion Auth

You load just Ion Auth like an other library:

{% highlight php %} $this->load->library('ion_auth'); {% endhighlight %}

You can also autoload the library By setting the Ion Auth library in your autoload libraries in application/config/autoload.php.

{% highlight php %} $autoload['libraries'] = array('ion_auth'); {% endhighlight %}

Configuration options

Ion Auth is extremely configurable. The following configuration option are avaible.
To change configuration options simply edit the config/ion_auth.php file.
Edit the ion_auth $config array only as needed.

$config['tables']['groups']
The table name to use for the groups table. DEFAULT is 'groups'.

$config['tables']['users']
The table name to use for the users table. DEFAULT is 'users'.

$config['tables']['users_groups']
The table name to use for the users groups table. DEFAULT is 'users_groups'.

$config['tables']['login_attempts']
The table name to use for the login attempts table. DEFAULT is 'login_attempts'.

$config['site_title']
The title of your site, used for email.

$config['admin_mail']
Your administrator email address. DEFAULT is 'admin@example.com'.

$config['default_group']
Name of the default user group. DEFAULT is 'members'.

$config['admin_group']
Name of the admin group. DEFAULT is 'admin'.

$config['join']['users']
Users table column you want to join with. DEFAULT is 'user_id'.

$config['join']['groups']
Group table column you want to join with. DEFAULT is 'group_id'.

$config['identity']
Column to use for uniquely identifing user/logging in/etc. Usual choices are 'email' OR 'username'.
You should add an index in the users table for whatever you set this option to. DEFAULT is 'email'.

$config['min_password_length']
Minimum length of passwords. DEFAULT is '8'.

$config['max_password_length']
Maximum length of passwords. DEFAULT is '20'.

$config['email_activation']
TRUE or FALSE. Sets whether to require email activation or not. DEFAULT is 'false'.

$config['remember_users']
TRUE or FALSE. Sets whether to enable 'remember me' functionality or not. DEFAULT is 'true'.

$config['user_expire']
Sets how long to remember the user for in seconds. Set to zero for no expiration. DEFAULT is '86500'.

$config['user_extend_on_login']
TRUE or FALSE. Extend the users session expiration on Login. DEFAULT is 'false'.

$config['email_type']
Email content type. DEFAULT is 'html'.

$config['email_templates']
Folder where the email view templates are stroed. DEFAULT is 'auth/email/'.

$config['email_activate']
Filename of the email activation view template. DEFAULT is 'activate/tbl.php'.

$config['email_forgot_password']
Filename of the forgot password email view template. DEFAULT is 'forgot_password.tpl.php'.

$config['email_forgot_password_complete']
Filename of the forgot password complete amail view template. DEFAULT is 'new_password/tpl.php'

$config['salt_length']
Length of the encryption salt. DEFAULT is '10'.

$config['store_salt']
TRUE or FALSE. Store the salt in a separate database column or not.
This can be useful for interating with existing apps. DEFAULT is 'false'.

$config['forgot_password_expiration']
Number of seconds before a forgot password request expires. If set to 0, requests will not expire. DEFAULT is 0.

$config['track_login_attempts']
Track the number of failed login attampts for each user or IP. DEFAULT is 'false'.

$config['maximum_login_attempts']
Set the maximum number of failed login attampts. This maximum is not enforced by the library,
but is used by $this->ion_auth->is_max_login_attempts_exceeded(). The controller should check
this function and act appropriately. if set to 0, there is no maximum. DEFAULT is 3.

$config['message_start_delimiter']
Starting delimiter for messages. DEFAULT is '<p>'.

$config['message_end_delimiter']
Ending delimiter for messages. DEFAULT is '</p>'.

$config['error_start_delimiter']
Starting dilimiter for errors. DEFAULT is '<p>'.

$config['error_end_delimiter']
Ending dilimiters for errors. DEFAULT is '</p>.'
